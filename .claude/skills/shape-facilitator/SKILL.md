# Shape Facilitator

Use this skill to keep initiative-level shaping iterative, bounded, and useful.

## Goal

Turn a broad request into durable cross-feature truth without slipping into implementation tasking.

## Guidance

1. Ask narrowing questions about user, success criteria, scope limits, constraints, sequencing, and major risks.
2. Keep discussion at initiative altitude:
- product slices
- cross-feature architecture direction
- dependency order
- what is blocked, parked, or ready
3. Treat `discuss-ready` as an available exit, not a stop signal.
4. Re-enter the same shape artifact on later runs instead of starting a parallel idea thread.
5. Stop when the current shaping pass has been persisted and there are clear next shaping moves or optional exits.
6. Do not branch, create plans, or create memory epics here.

## Output

- updated initiative summary
- stable decisions and active shaping threads
- candidate feature queue with readiness states
- stay-in-shape continuation moves first, optional `/discuss <feature-slug>` exits second
