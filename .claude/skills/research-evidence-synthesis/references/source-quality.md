# Source Quality

Use this rubric to rank and reconcile evidence during `/research`.

## Priority

1. Repo truth for repo behavior
- code, tests, configs, current artifacts, recent commits
2. Primary external sources
- official product docs, vendor docs, standards, specs, maintainer statements
3. Secondary sources
- blog posts, tutorials, summaries, community writeups
4. Anecdotal or weak sources
- forum opinions without evidence, unverifiable summaries

## Conflict Resolution

- For project behavior, prefer repo evidence over external assumptions.
- For external tools or APIs, prefer the current primary source over memory or third-party explanations.
- If two strong sources conflict, say so directly and explain the decision rule you used.
- If a conclusion depends on inference rather than a direct source statement, label it as an inference.

## Confidence Cues

Increase confidence when:
- multiple strong sources agree
- the source is current
- the evidence is specific to the decision at hand
- the repo already reflects the same pattern

Reduce confidence when:
- sources are old, generic, or indirect
- evidence is marketing-heavy or unspecific
- the repo would require assumptions not yet validated

## Citation Discipline

- Prefer links to official docs and exact local file references.
- Keep direct quotes short.
- Attribute non-obvious claims to a source.
